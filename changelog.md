# 版本日志

## v1.0.0 (2026-02-27)

### 🎉 首次发布

基于 [picgo-plugin-lankong](https://github.com/hellodk34/picgo-plugin-lankong) 重构，全新发布 `picgo-plugin-lskypro`。

### ✨ 新特性

- ✅ 支持兰空图床 V1 和 V2 版本
- ✅ 支持自定义存储策略
- ✅ 支持相册分类（V2）
- ✅ 支持图片权限设置（公开/私有）
- ✅ 支持同步删除功能（V2）
- ✅ 支持忽略证书错误
- ✅ 自动设置默认存储策略（strategy_id=1）
- ✅ **自动处理 Token 格式**：无需手动添加 `Bearer` 前缀，插件自动处理
- ✅ **优化配置界面**：所有字段都有详细的中文说明和使用提示
- ✅ **Strategy ID 获取指引**：配置界面直接提示如何获取存储策略 ID

### 🐛 修复

- 修复上传失败问题：当用户未填写 strategy_id 时自动使用默认值 1
- 优化删除提示信息，说明删除操作可能存在延迟

### 💡 用户体验改进

- Token 字段自动添加 `Bearer` 前缀，用户只需粘贴原始 Token
- Strategy ID 默认值设为 `1`，并在配置界面提供获取方法说明
- 所有配置项都有清晰的中文说明和示例
- 版本选择默认为 V2（推荐新用户使用）

### 📝 文档

- 完整的中文使用文档
- 详细的配置说明
- 常见问题解答
- API 接口文档
- Strategy ID 获取方法详解

### 🔧 技术改进

- 重构项目结构
- 优化代码质量
- 添加 TypeScript 类型定义
- 改进错误处理机制
- 智能 Token 格式处理

## 历史版本（基于原项目）

### v1.1.3 (2022-08-20)
- 跟进兰空图床付费版根据相册 id，权限上传图片的 api 功能
- 新增 PicGo lankong 插件的相关设置选项并提供功能

### v1.0.6 (2022-04-01)
- 修复同步删除的 bug

### v1.0.5 (2022-03-30)
- 增加同步删除功能，仅支持 V2

### v1.0.4 (2022-03-30)
- 版本选择改成下拉菜单样式
- https agent 代理对象的创建放在分支条件中增加程序性能

### v1.0.3 (2022-03-26)
- 表单增加存储策略 ID 录入，仅供 V2 用户使用
- V1 以及 V2 使用默认存储策略时请留空

### v1.0.2 (2022-03-23)
- 增加 `Lsky Pro Version` 开关，支持兰空图床 V2
- 开关默认关闭，关闭代表 V1，打开即可使用 V2 版本的兰空图床 API 上传

### v1.0.1 (2022-03-22)
- 支持兰空图床 V1 的版本正式发布
